프로젝트 시작 배경

네이버 부스트캠프에서 팀 프로젝트를 진행하면서, 클라우드 인프라 구축의 복잡성을 체감했습니다. Terraform 코드를 직접 작성하는 것은 초보자에게 진입장벽이 높고, 인프라 구조를 시각적으로 파악하기 어려웠습니다. "드래그 앤 드롭으로 인프라를 설계하고, 자동으로 Terraform 코드가 생성된다면 어떨까?"라는 아이디어에서 시작되었습니다.

맡은 역할

  • 백엔드 아키텍처 설계: NestJS 기반 API 서버 구축
  • Terraform 컨버터 개발: 프론트엔드에서 받은 JSON 데이터를 실제 Terraform 코드로 변환하는 핵심 로직 담당
  • 성능 최적화: 캐싱 메커니즘과 Strategy 패턴을 도입하여 변환 속도 70% 향상

주요 도전과 해결

  • 문제: 프론트엔드와 백엔드에서 동일한 검증 로직이 중복 구현되어 유지보수가 어려웠음
    해결: Zod 라이브러리를 활용해 공통 검증 패키지를 분리하고, 두 환경에서 동일한 스키마를 사용하도록 개선
  • 문제: 새로운 클라우드 리소스 추가할 때마다 거대한 switch문을 수정해야 했음
    해결: Strategy & Factory 패턴을 도입하여 새로운 리소스 추가 시 코드 수정량을 40-70% 줄임
TypeScript NestJS AWS S3 MySQL

기술 스택 선택 이유

  • TypeScript + NestJS
  • • 타입 안정성: Terraform 코드 생성 시 복잡한 리소스 구조를 다루기 때문에 컴파일 타임 타입 검증이 필수적입니다.
  • • 확장성: 다양한 클라우드 프로바이더 지원을 위한 모듈화된 구조 설계가 용이합니다.
  • • 개발 생산성: 데코레이터 기반 의존성 주입으로 테스트 가능한 코드를 쉽게 작성할 수 있습니다.
  • MySQL
  • • 관계형 데이터 관리: 리소스 간 복잡한 참조 관계를 효율적으로 관리할 수 있습니다.
  • • ACID 보장: Terraform 상태 관리의 일관성을 보장합니다.
  • • 팀 익숙도: 팀원들이 MySQL에 익숙해 개발 속도가 향상됩니다.
  • Zod 검증 라이브러리
  • • 공통 패키지화: 프론트엔드와 백엔드에서 동일한 검증 로직을 공유할 수 있습니다.
  • • 타입 추론: TypeScript와 완벽하게 호환되어 런타임 안전성을 보장합니다.
  • • 스키마 기반 검증: 클라우드 리소스의 복잡한 속성 검증에 적합합니다.

프로젝트 시작 배경

혼자 자취하면서 "냉장고에 있는 재료로 뭘 만들어 먹지?"라는 고민을 자주 했습니다. 기존 레시피 앱들은 완성된 요리 중심이었는데, "있는 재료로 만들 수 있는 요리"를 찾는 역방향 검색 서비스가 있으면 좋겠다고 생각했습니다.

맡은 역할

  • 백엔드 개발 전담: 3인 팀에서 백엔드 아키텍처 설계부터 API 구현까지 담당
  • 외부 API 연동: 공공데이터포털의 레시피 API 연동 및 데이터 가공
  • 성능 최적화: 검색 성능 향상과 이미지 처리 시스템 개선

인상 깊었던 문제 해결

  • 문제: 레시피 이미지 업로드 시 파일명 충돌로 이미지가 덮어씌워지는 문제가 발생
    해결: UUID를 활용한 고유 파일명 생성 시스템을 도입하여 충돌 가능성을 0%로 만듦
  • 문제: 레시피 검색 시 대소문자 구분으로 인해 검색 성공률이 낮았음
    해결: JPQL의 LIKE와 LOWER 함수를 조합하여 유연한 검색 시스템을 구현, 검색 성공률을 45%에서 95%로 향상
Java Spring Boot PostgreSQL AWS S3 OAuth2 JWT

기술 스택 선택 이유

  • PostgreSQL
  • • JSON 지원: 레시피의 복잡한 메타데이터(재료, 단계별 설명) 저장에 최적입니다.
  • • 전문 검색: 레시피명 검색 성능 최적화를 위한 고급 인덱싱 기능을 제공합니다.
  • AWS S3
  • • 무제한 확장성: 레시피 이미지 용량 증가에 대한 무제한 확장이 가능합니다.
  • • 비용 효율성: 소규모 프로젝트에서 서버 스토리지 대비 비용을 절약할 수 있습니다.

관련 기술 블로그

  • • JPA @ElementCollection으로 레시피 데이터 모델링 개선하기
  • • JPA 검색 쿼리 최적화로 검색 성능 개선하기
  • • UUID를 활용한 다중 이미지 업로드 시스템 개선기
